# Claude Instructions for argocd-scaffolding

## README hygiene

After **any** change to the following, update `README.md` to stay in sync:
- Directory structure (new dirs, renamed dirs, removed dirs)
- Key technologies (added/removed tools or frameworks)
- Workflow steps (new tenant, new cluster, deploying apps)
- Scripts (`scripts/`) — purpose, when to run, inputs/outputs
- Templates (`templates/`) — structure and usage
- GitHub Actions workflows (`.github/workflows/`) — triggers and behavior
- Terraform modules or layout changes

Do not leave `README.md` describing a state that no longer exists in the repo.

## Generated files — do not edit manually

The following files are regenerated by `make generate` and must not be edited by hand:

| File | Generated by |
|------|-------------|
| `terraform/bootstrap/providers.tf` | `scripts/generate-bootstrap.py` (renders via Jinja2 `templates/bootstrap/providers.tf.j2`) |
| `terraform/bootstrap/main.tf` | `scripts/generate-bootstrap.py` (renders via Jinja2 `templates/bootstrap/main.tf.j2`) |
| `argocd/projects/*.yaml` | `terraform/generate` (renders via `templatefile()` `templates/tenant/project.tftpl`) |
| `argocd/projects/kustomization.yaml` | `terraform/generate` (renders via `templatefile()` `templates/tenant/kustomization.tftpl`) |
| `infrastructure/clusters/*/vars/kustomization.yaml` | `terraform/generate` |

Post-Terraform generated files (written during `make apply-infra`):

| File | Generated by |
|------|-------------|
| `infrastructure/clusters/*/vars/tenant-vars.yaml` | `terraform/infra/generate-details.tf` (renders via `templatefile()` `templates/tenant/vars.tftpl`) |
| `infrastructure/clusters/*/*/namespace-details.yaml` | `terraform/infra/generate-details.tf` (renders via `templatefile()` `templates/tenant/namespace-details.tftpl`) |

## Source of truth files — edit these, not the generated output

| File | Controls |
|------|---------|
| `terraform/infra/tenants.yaml` | Tenants, namespaces, ArgoCD bootstrap |
| `infrastructure/clusters/{tenant}/{namespace-id}/{cluster}/kustomization.yaml` | Per-cluster components and feature flags |
| `argocd/repo-config.yaml` | Single repo URL used by all ApplicationSets |

## Cluster directory structure

Clusters live at `infrastructure/clusters/{tenant}/{namespace-id}/{cluster}/` where:
- `{namespace-id}` is the exact Terraform-generated VCFA supervisor namespace name (e.g. `dev-1-4gqxg`) — created by `make apply-infra`
- `{cluster}` is the human-chosen cluster name

Each namespace directory contains `namespace-details.yaml` (Terraform-generated). Each cluster directory contains `kustomization.yaml` — the **only** file requiring a human-set value (`cluster_name` in the `configMapGenerator`).

## Workflows

### Adding a new tenant
1. Add entry to `terraform/infra/tenants.yaml`
2. Run `make generate` (or push — GitHub Actions handles it)
3. Run `make apply` to provision infra + bootstrap ArgoCD

### Adding a new cluster
1. Run `make apply-infra` to ensure the namespace directory exists at `infrastructure/clusters/{tenant}/{namespace-id}/`
2. Copy `templates/cluster/` to `infrastructure/clusters/{tenant}/{namespace-id}/{cluster}/`
3. Edit the ONE value in `kustomization.yaml`: set `cluster_name={cluster}` in the `configMapGenerator` literal
4. Commit — ArgoCD ApplicationSet picks up the new cluster directory automatically

### After `make apply-infra`
Terraform automatically writes `namespace-details.yaml` and `tenant-vars.yaml` into the cluster tree. No separate generate step needed — commit the new/updated files.
