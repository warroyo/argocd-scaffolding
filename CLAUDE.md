# Claude Instructions for argocd-scaffolding

## README hygiene

After **any** change to the following, update `README.md` to stay in sync:
- Directory structure (new dirs, renamed dirs, removed dirs)
- Key technologies (added/removed tools or frameworks)
- Workflow steps (new tenant, new cluster, deploying apps)
- Scripts (`scripts/`) — purpose, when to run, inputs/outputs
- Templates (`templates/`) — structure and usage
- GitHub Actions workflows (`.github/workflows/`) — triggers and behavior
- Terraform modules or layout changes

Do not leave `README.md` describing a state that no longer exists in the repo.

## Generated files — do not edit manually

The following files are regenerated by `make generate` and must not be edited by hand:

| File | Generated by |
|------|-------------|
| `terraform/bootstrap/providers.tf` | `scripts/generate-bootstrap.py` |
| `terraform/bootstrap/main.tf` | `scripts/generate-bootstrap.py` |
| `argocd/projects/*.yaml` | `scripts/generate-infra.py` (renders via ytt `templates/tenant/project.yaml`) |
| `argocd/projects/kustomization.yaml` | `scripts/generate-infra.py` (renders via ytt `templates/tenant/kustomization.yaml`) |
| `infrastructure/clusters/*/vars/kustomization.yaml` | `scripts/generate-infra.py` |
| `infrastructure/clusters/*/*/kustomization.yaml` | `make generate` (ytt) |
| `infrastructure/clusters/*/*/apps/kustomization.yaml` | `make generate` (ytt) |
| `infrastructure/clusters/*/*/cluster-details.yaml` | `make generate` (ytt), then `make generate-details` fills `namespace`/`argo_namespace` |

Post-Terraform generated files (require state):

| File | Generated by |
|------|-------------|
| `infrastructure/clusters/*/vars/tenant-vars.yaml` | `scripts/generate-details.py` |
| `infrastructure/clusters/*/*/cluster-details.yaml` (namespace fields) | `scripts/generate-details.py` |

## Source of truth files — edit these, not the generated output

| File | Controls |
|------|---------|
| `terraform/infra/tenants.yaml` | Tenants, namespaces, ArgoCD bootstrap |
| `infrastructure/clusters/{tenant}/{cluster}/cluster-values.yaml` | Per-cluster feature flags and config |
| `argocd/repo-config.yaml` | Single repo URL used by all ApplicationSets |

## Workflows

### Adding a new tenant
1. Add entry to `terraform/infra/tenants.yaml`
2. Run `make generate` (or push — GitHub Actions handles it)
3. Run `make apply` to provision infra + bootstrap ArgoCD

### Adding a new cluster
1. Create `infrastructure/clusters/{tenant}/{cluster}/cluster-values.yaml`
2. Run `make generate` (or push — GitHub Actions handles it)
3. Commit generated files — ArgoCD ApplicationSet picks up the new cluster automatically

### After `make apply-infra`
Run `make generate-details` (called automatically by `apply-infra`) to populate auto-generated namespace IDs and tenant UUIDs into the GitOps files.
